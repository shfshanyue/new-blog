# 微信图文消息写作规范

## 适用场景

用户要求写或润色微信图文消息、公众号推文、微信图文内容，或明确按本规范输出时，遵循以下规则。

## 保存位置与 Frontmatter

- 内容保存到 `src/content/wechat/` 目录，文件名使用英文短横线命名（如 `my-article-title.md`）。
- 文件顶部使用 Frontmatter：
  - `title`：简洁明确的标题
  - `description`：80–120 字的摘要，概括正文要点
  - `pubDatetime`：发布时间，使用 ISO 8601 格式（如 `2025-01-31T00:00:00Z`）
  - `tags`：3–5 个标签，便于分类检索，如 `["标签一", "标签二"]`

## 正文格式

1. 不使用 markdown 符号：禁止使用加粗、反引号、标题符号、分割线等（如 **、`、##、---）。
2. 尽量不使用数字编号，除非内容较多、确实需要条理化时才用。
3. 不插入图片，纯文字呈现。
4. 每一段后空一行（段末换行后再换行一次）。
5. 正文末尾添加 #全栈成长之路。

## 篇幅与字数

- 正文以**汉字**为主计量，宜维持在 **300 字左右**（约 250–400 字可接受）：太短易显单薄，太长不像单条图文素材。
- 统计时去掉 Frontmatter，以正文段落为准；话题行（如 `#全栈成长之路`）里的汉字可计入总篇幅，也可单独一行不计入——以成稿读起来节奏紧凑为准。
- 文中的英文、数字、域名或文件名（如 `agents.md`）不占汉字额度时，仍须控制整体阅读长度，避免正文实际过长。

## 行文风格（从现有文章中总结）

以下风格特征从 `src/content/wechat/` 目录中的真实文章提炼而来，写作时须对照执行。

### 1. 开篇：一句话切入，不铺垫

- 第一段直接亮出话题或场景，一两句话说完，不要有「随着……的发展」「在当今……的时代」式的宏大开场。
- 常见开法：
  - 抛一个问题（「你们公司报销 Token 吗？」）
  - 讲一个刚发生的事（「今天简单试了下 Amp」「gpt-5.4-mini 和 gpt-5.4-nano 今天发布了」）
  - 转述群里/身边的见闻（「我的技术群里有个哥们儿吐槽」「有群友提问」）

### 2. 语气：聊天式，像跟同事随口说

- 用「我」「你」「我们」拉近距离；允许出现口语词（「牛逼」「擦屁股」「划划水」「爽」「肉疼」）。
- 可以夹杂调侃、自嘲（「对他被批评表达同情，并对他可以无限使用 Opus 表达羡慕」）。
- 像真人聊天：长短句自然穿插，有停顿、有呼吸，别一逗到底写成报告体。
- 不用「此外」「综上所述」「毋庸置疑」等书面过渡词。
- 语气不必每篇都带情绪色彩——「平叙型」同样合法：平铺直叙地陈述工具、用法、决策，不用刻意放大情绪或加情绪词，读起来像随口跟朋友说「我现在用的是这几个」即可。

### 3. 论证：讲具体细节，不讲大道理

- 用真实数字、工具名、模型名说事（「Sonnet 每百万 token 3 美元」「220MB 的 zip」「60MB 的图片」）。
- 多用动词和名词，少用形容词堆砌。
- 不做空泛升华（不写「AI 正在深刻改变我们的工作方式」这种话）。
- 允许举自己或群友的亲身经历当例子。

### 4. 结构：自然段落 + 可选编号，不用标题层级

- 正文由若干自然段组成，用空行分隔，不用 markdown 标题。
- 尽量不用编号，靠段落自然推进；只有内容较多、确实需要条理化时才用「1 xxx」「2 xxx」纯数字编号（无圆点、无括号）。
- 用编号时条目不宜过多，通常 2–4 条，每条一两句话即可。

### 5. 收尾：落点明确，不喊口号

- 最后一段通常是一句总结、一个建议、或一个反问，不超过两句话。
- 不写「让我们一起拥抱……」「未来已来」「值得每一个人深思」之类的口号。
- 结尾后紧跟 `#全栈成长之路`（作为话题标签独占一行，前面空一行）。

### 6. 标题（title）风格

- 口语化，像朋友圈/群聊里的一句话，带情绪或悬念。
- 常见模式：
  - 带判断/观点（「以后 Code Review 会比写代码更重要」）
  - 带情绪词（「试用了一小时 Amp，比 Cursor 爽太多了」「比没额度更慌的，是额度没用完」）
  - 带疑问（「公司不报销 Token，为什么你反而更累了」）
  - 比喻/梗（「猴子拿着冲锋枪」）
- 不要用冒号分隔式标题（如「深度解析：xxx 的五大趋势」），太正式。

### 7. 描述（description）风格

- 80–120 字，是正文的浓缩版，能独立阅读。
- 用具体信息而非抽象概括：提及工具名、数字、场景。
- 语气与正文一致，不要变成「本文将为您介绍……」的摘要腔。

## 润色时的文风要求

润色或改写时，生成内容不要 AI 风。具体做到：

- 少用或不用「首先、其次、此外、综上所述、毋庸置疑、在这个……的时代」等套话与模板式过渡。
- 避免排比堆砌、空洞升华、过度对称的句式；口语化表达，保留个人语气，句子长短随意，像随口说出来的。
- 用具体细节替代抽象概括，用动词和名词替代形容词堆砌。
- 整体读起来像人写的、可随口说出来的话，而非机翻或范文腔。
