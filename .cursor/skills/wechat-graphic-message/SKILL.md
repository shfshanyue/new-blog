# 微信图文消息写作规范

## 适用场景

用户要求写或润色微信图文消息、公众号推文、微信图文内容，或明确按本规范输出时，遵循以下规则。

## 正文格式

1. 不使用 markdown 符号：禁止使用加粗、反引号、标题符号、分割线等（如 **、`、##、---）。
2. 可使用 1 2 3 等数字编号表达条理。
3. 不插入图片，纯文字呈现。
4. 每一段后空一行（段末换行后再换行一次）。
5. 正文末尾添加 #全栈成长之路。

## 润色时的文风要求

润色或改写时，生成内容不要 AI 风。具体做到：

- 少用或不用「首先、其次、此外、综上所述、毋庸置疑、在这个……的时代」等套话与模板式过渡。
- 避免排比堆砌、空洞升华、过度对称的句式；多用短句、口语化表达，保留个人语气。
- 用具体细节替代抽象概括，用动词和名词替代形容词堆砌。
- 整体读起来像人写的、可随口说出来的话，而非机翻或范文腔。

## 输出示例（结构示意）

```
正文第一段。

正文第二段。

1 第一条要点。

2 第二条要点。

3 第三条要点。

#全栈成长之路
```

## 自检清单

- [ ] 未使用 **、`、##、--- 等 markdown 符号
- [ ] 仅用 1 2 3 等数字编号，未用其他格式强调
- [ ] 未插入图片
- [ ] 每一段后均空一行
- [ ] 正文末尾已添加 #全栈成长之路
- [ ] 润色时未出现 AI 风套话、排比堆砌或范文腔
