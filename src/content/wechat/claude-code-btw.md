Claude Code 很早出了个小功能，叫 /btw。

以前项目构建的时候，代码跑起来可能要十几分钟，这段时间正好去倒杯水、透透气。现在 AI 写代码的等待时间更长，而且频率比以前构建高多了——我们反而容易拿起手机刷红果短剧，或者打开 Bilibili，一刷就是一段时间。

/btw 给我解决了这个问题，那就可以回顾一下刚才与 AI 的决策，以及 AI 的思路。

好消息是，Cursor CLI 现在也有这个功能了，不用再只靠 Claude Code 才能享受这个体验。另外，Cursor CLI 目前也支持粘贴剪切板中的图片了。
