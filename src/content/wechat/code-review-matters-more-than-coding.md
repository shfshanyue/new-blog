我越来越觉得，以后 Code Review 能力会比写代码能力更重要。

写代码这件事，AI 已经能帮很多忙了。但看懂一段代码有没有问题、能不能在复杂系统里快速定位隐患，这种「眼力」和「判断」短期内很难被完全替代。

所以面试很可能跟着变：不再让你现场手写一道算法，而是给你一段可能有小瑕疵的代码，让你去挑毛病、说清楚哪里不对、为什么不对。

1 从「能写多少行」到「能把控多复杂」。

以前大家会下意识用代码量来衡量产出，以后会更看重：你负责的模块边界清不清楚、依赖关系能不能说清楚、出问题时能不能很快缩小范围。这些本质上都是「读代码、审代码、控系统」的能力。

2 眼力比手速值钱。

能一眼看出边界条件没覆盖、能发现并发下的竞态——这类能力会越来越值钱。写可以交给 AI 起完成，但「这个决策方案该不该上」的判断，还是要人来扛。

以后面试说不定真就改成：发你一段代码，限时找问题。
