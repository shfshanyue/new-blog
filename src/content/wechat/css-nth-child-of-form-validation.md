以前做多步表单，校验状态基本全靠 JS 手搓。有几个无效字段、错误提示要不要展示、提交按钮该不该灰，全是自己维护的状态。

细想一下，这种活儿挺机械的，但一碰上动态增删行就容易写出一堆监听器。

最近看到有人用纯 CSS 搞这块，思路挺有意思：nth-child 配上 of 选择器，再叠一层 has()，就能在表单里数清楚哪些字段有效、哪些无效，连错误汇总区和按钮禁用态都能带上。

跟裸用 invalid 比，按类型分组统计的时候更稳，新增一行也不用去改 JS 里的计数逻辑。

不过别想全靠它扛复杂业务。浏览器支持要对一下目标版本，无障碍还得配 aria-live，真有复杂联动最后还是得 JS 收尾。

当渐进增强倒是挺香，底子在那，脚本抽风了也不至于完全裸奔。
