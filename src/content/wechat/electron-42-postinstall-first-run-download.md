Electron 升到 42 这一档，有个体验上的变化很直观：你跑 npm install 的时候，以前那种 postinstall 里吭哧吭哧下几百兆二进制的感觉，轻了很多。

原因是策略改了：安装阶段不再顺手把运行时拉下来，改到用户第一次真正打开应用时再下。对本地开发来说，冷启动会多一点变量，但装依赖这条链路确实更干净了。

我把它和最近大家聊 npm 那堆事放在一起看，其实是一条线上的：生命周期脚本一动就是远程拉东西、跑脚本，爆炸半径确实大。包管理器那边也开始更谨慎，桌面侧老牌方案跟着收敛，不奇怪。

当然这不等于你可以偷懒不管供应链。升级 Electron 仍然要把原生模块 ABI、自动更新、签名分发都跑一遍，别只改个版本号就当完事。

你身边有没有团队因为这个改动，把 CI 的时间线也重写了一遍？
