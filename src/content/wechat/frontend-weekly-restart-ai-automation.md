2021 年我做过一个前端周报，更了三十多期，后来精力实在跟不上就停了。

2024 年借着 GPT 又捡起来跟了十几期。那时候的 AI 工具还比较原始，每期还是要花 1 到 2 个小时。现在打算重新做起来，因为成本已经低很多了。

有只龙虾的话，它能自动帮你整理好；没有龙虾，流程也比以前简单。

现在的流程可以这样跑：

1 自动化处理。OpenAI Codex 有 Automation，Cursor 的 Cloud Agent 也有 Automation，可以交给它们定时任务去跑，并提交 PR。

2 任务安排。设定每周三发布，那就让它在每周二自动整理好并提交一个 PR。

3 人工审核。自己再做一轮审核，哪里不合理就做 Code Review。

4 闭环整改。Automation 根据 Code Review 意见去改，改完再提交，如此往复。

算下来，从最早一期要 4 小时，到后来缩到一个半小时，到现在可能只需要半小时。

而且这半小时多半花在审核以及决策上，真正手动处理那些枯燥事情的时间很少。这么一想，周报确实可以重新捡起来了。
