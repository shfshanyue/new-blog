HeyGen 把 HyperFrames 开源了，说白了就是用 HTML 和 JavaScript 去拼一段视频，跟 Remotion 有点像，但更啃原生 DOM 和一些现成块，还能把已有音视频轨叠进去。

我第一反应是：团队要是只想录个更新说明、产品演示、落地页动效，不一定非要整栈上 React 视频管线。你页面里现成的组件、路由、动效，很多时候就是镜头本身，导出成 mp4 丢给运营发社媒或塞进 release note，链路会短一截。

坑也实在：无头跑的时候字体加载、图片外链、动画掉帧都得提前想清楚，复杂运镜别指望开箱即神。更合理的用法是先把「一个稳定页面 + 固定旁白 + 固定分辨率」跑通，再考虑系列化。

如果你本来就要录屏做物料，可以从最短那条片子试点，省下来的维护成本才是真的爽。
