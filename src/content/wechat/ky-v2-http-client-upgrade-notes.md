我最近又在项目里翻 HTTP 封装，顺手看了眼 ky 的大版本。

说实话，这种库升级最怕的不是 API 名字变了，而是你根本不知道团队里谁在偷偷依赖旧行为。比如以前 searchParams 覆盖输入 URL，现在改成合并，这种细节一不注意就会在预发里炸成 404 或者查参串味。

2.0 这条线把 Node 抬到 22，对我来说倒是好事，反正新机器都往 LTS 靠。totalTimeout 这种跨重试的总时长也蛮实在，省得你自己在重试逻辑里再套一层计时器。

但我也劝一句别跟风：如果你只是想要个比原生 fetch 好写一点的客户端，升级前先 grep 一下项目里 ky 的钩子和 prefix 用法，改完跑一遍你最肉疼的那几个接口用例，比读完十篇推文都管用。
