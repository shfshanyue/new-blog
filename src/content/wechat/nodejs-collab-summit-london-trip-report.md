四月份在伦敦办了今年第一场 Node 协作峰会，彭博出的场地，现场四十来号人，线上还有十几个。官方博客把录音和议题摊开，读完像在看维护者给自己写的备忘录。

从 Node 27 起要换发布节奏，主版本号对齐它第一次进 Current 那一年，一眼读出是哪一年的线，同时少开几条并行维护线，省人力打安全补丁。现在四五条线每次漏洞都跟，志愿维护扛不住。

James Snell 讲流，node stream 和 Web Streams 并了很久，现在往 async await、for await 收，缓冲满了要你把背压写清楚，不再装黑盒。

聊 AI 更真实：会前代理写的大 PR 把 VFS 弄进核心吵翻，会上过了 OpenJS 法律意见和别的项目咋处理。大改动先设计文档、RFC、维护者点头，人兜底质量。另一头更烦，赏金驱动下 AI 批量吐高危报告，重复没复现还把普通 bug 吹成核弹，triage 被泡爆，团队在聊流程要不要更公开，别让 embargo 卡 CI。

跑 Node 别光盯小版本，这篇 trip report 当雷达扫一遍，比刷信息流省事。
