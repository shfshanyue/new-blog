我最近在翻 OWASP 那份 npm 安全速查，里面有一条写得特别实在：别默认信任所有 lifecycle script，尤其是装包时自动跑的 postinstall。

你想想，npm 装依赖本质上是在你机器上执行别人仓库里的脚本，一旦包被投毒或者名字打错装成仿冒包，最先动手脚的就是这类钩子。团队里新人多、依赖更新又勤的时候，这条线真的不夸张。

我现在在 CI 里会习惯性先走 npm ci 并带上忽略脚本的策略，只在确实需要编译原生模块的那几步再单独放开，相当于把「能跑脚本的口子」收窄。再配合清单里提到的可信发布、依赖名仔细核对、别随手拼包名，工具链的攻击面会小一截。

当然这不是万能药，本地开发该装还是要装，但流水线是批量复制、权限往往还不低，先从流水线收紧，通常比事后 grep 日志找谁执行了奇怪命令要划算。

你那边要是也遇到过装完依赖机器变慢的怪事，不妨从这一条开始排查。
