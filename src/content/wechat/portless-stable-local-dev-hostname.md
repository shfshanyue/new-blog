你还在 README 里写「先起服务再去找端口」吗？我自己都嫌烦，新人机器端口被占用又要改环境变量，群里还得截图对照谁是多少。

Vercel Labs 开源了个叫 portless 的小工具，思路是把本地入口从一串 localhost 加数字端口，换成更像正式环境的名字，挂在 localhost 下面那种固定主机名，还带 HTTPS。对人来说少背随机数，对协作来说大家看到的是同一套入口，文档里也少写「我这边 3001 你那边 3000」这种补丁句。

如果你用 Tailscale 连远程开发机，新版本在这块也做了体验强化，入口不那么飘。

真要落地别只顾本地爽：OAuth 回调域名要不要跟着换、公司反向代理有没有额外规则、浏览器 HSTS 预加载会不会把你卡在证书或域名上，联调前一天就对一遍。

最近在整理新人上手文档的话，可以把它当成第一印象优化项试一圈，至少少几次「你那边端口多少」的废话。
