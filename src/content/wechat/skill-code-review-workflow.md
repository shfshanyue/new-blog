上次聊过用最佳实践类的 Skill 拉高自己跟项目的水平，这次说下具体怎么操作。

你先 @ 一个和项目技术栈对应的 Skill，比如 Vue、React、Next.js、PostgreSQL、Redis、Vite、SEO、a11y、Performance、Security 之类的，让它对你的代码做一次审查。这里有个关键：不要让它直接改，只让它把审查结果写出来，存到你指定的目录里。比如建一个 code_review 目录，按「日期 + 这次审了什么 + 序号」来命名文件，每次的审查报告都单独存一份。

Skill 会按条理列出一堆修改建议，并标触发规则以及重要程度。接下来是你自己过一遍。

1 要修的东西，一条一条指给 AI，让它一条一条改。你重点看它具体改了什么、怎么改的。

2 对还不熟的那块，就一条一条慢慢来，借机把每一条都搞懂。

3 对已经比较熟的内容，可以挑几条合并在一起，让 AI 一次性改，省时间。

这样既能把审查结果沉淀下来，又能在修的过程中观察 AI 的修复动作，该慢的慢、该批量的批量，节奏自己控。
