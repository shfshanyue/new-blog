分享最近使用 Skill 的一些想法

多用一些编码的最佳实践，比如 react、vue、next、postgresql，每次使用 cursor 或 claude code 创建好 Plan 之后，显式地 @skill 去编码。如果前期编码没有使用到的话，可以 @skill 让它去做 code review。

但这其实本质上来说是 Prompt，只不过将最佳实践作为了 Prompt。比如我使用 cursor 写好博客，我需要给文章配图，那这个时候就想让 AI 按照文章内容进行配图，但是 cursor 没有生成图片的能力，这个时候应该怎么办。

以前的话，就有可能调用一些生成图片的 MCP，但是不太好用。但现在就可以写 skills，基于豆包生图或者 Banana 生图的 API 写一个 cli 工具，并在 skills 中附上 cli 工具的使用方法，我们在 @skills 就可以借助 cli 的能力生成文章对应的图片。

等实践了一段时间，我发现我们有两种使用 AI 生图的场景。第一是给文章配图，第二是生成文章的封面图。那我们就可以封装成两个 skill。并且在这两个 skill 中再让它调用生图的 skill。

这个时候，我们已经将文章写好了，并且可以发布在博客。但是我们想让他自动同步到各个平台，比如 x，以及 y 和 z 平台。此时就可以再写三个 skills，并且调用 agent-browser 的 skills 去自动登录发布。

再到后续，整个工程都可以做到自动化，每天定一个定时任务，每天八点 AI 自动总结今天热点，写好文章，自动发布各个平台。这样就可以批量制造 \_ \_ 内容了。话是这么说，不过发布这个事情最好还是由人来触发，毕竟连 CICD 最后部署生产环境的时候最好还是手动触发。

最后，在国内用豆包 API，在国外用 GPT API。
