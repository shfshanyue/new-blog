最近我的高级模型每个月 Token 都比较紧张，所以我做了一些节省 Token 的操作。

其中一个是关于 skill 的。我以前在全局目录下有 130 个 skill，像 Cursor 或者 CodeX 这些 AI 工具，都会去全局目录下加载它们的 name 与 description 等元数据作为上下文。但其实这 130 个我并不是都很需要，我一般流程中能用到的也就不到 10 个。

于是我就把多余的 120 个全删掉了。我大概算了一下：

1 这样一次估计能节省不到 1 万的 Token。

2 调用 100 次的话就是 100 万 Token，相当于节省了 15 美元。

现在能明显感觉到可用的 Token 比以前稍微多了一些，没有特别紧张。
