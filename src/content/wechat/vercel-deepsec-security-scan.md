我最近在翻安全工具，看到 Vercel 丢了个叫 deepsec 的开源项目出来，思路不是单纯堆规则，而是把静态扫描和编码 Agent 串成一条流水线。

你在仓库里跑完 init，它会先粗粒度把敏感区域圈出来，比如鉴权中间件、打款回调、读环境变量的地方。接着让能改代码那类 Agent 去跟调用链，看数据从哪进、写到哪、有没有漏校验。最后再开一轮 Agent 专门对着疑似点做二次确认，把明显误报压下去，免得 CI 里红一片你也不敢点忽略。

它支持在本机长跑，也能把任务 fan-out 到 Sandboxes 上做并行，适合开源后端或者特别肥的 monorepo 当增量体检。真要说短板，就是它再聪明也替代不了依赖审计和正经渗透，但当个常驻雷达、配合你们自己的鉴权模式写点 matcher，我觉得值回票价。

你要是也在头疼告警噪声，可以挑个非核心分支先跑一周，看误报率和覆盖面对不对胃口。
